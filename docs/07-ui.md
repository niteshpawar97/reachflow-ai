# ReachFlow AI — UI / UX Design

> Phase 8 deliverable. React + Vite SPA. Tailwind + shadcn-style components, TanStack Query, RHF+Zod,
> Recharts, Framer Motion. **Dark mode default**, fully responsive.

## Design principles
- **Operator-first:** you live in this tool daily → speed, keyboard shortcuts, dense-but-legible tables.
- **Human-in-the-loop is visible:** approve/edit AI drafts, review classifications — never a black box.
- **Deliverability always on screen:** mailbox/domain health is a persistent, glanceable signal.
- **Progressive disclosure:** simple by default, power on demand.

## Screens

### 1. Dashboard (home)
- KPI tiles: emails sent, delivered %, open %, reply %, meetings, deals value (range picker).
- **Deliverability strip:** mailbox health list (score, cap used today, warmup state), domain SPF/DKIM/DMARC.
- Conversion funnel (Recharts). Recent replies needing attention. Active campaigns snapshot. Cost widget (AI/scrape spend).
- Empty state → onboarding checklist (connect mailbox → import/find leads → create campaign).

### 2. Leads
- **Table** (virtualized): company, contact, country, industry, score (badge), status, audit ✓, verified ✓, actions.
- Filters: status, min score, industry, country, source, campaign. Saved views. Bulk-select → audit/verify/score/personalize/add-to-campaign.
- **Lead drawer/detail:** company info, **website audit report** (findings + AI summary + screenshot),
  score breakdown (factors), contacts, **personalization preview (editable, approve/reject/regenerate)**, activity.
- **Lead Finder tab:** search form (industry/geo/keywords/source) → job progress → results into leads.
- Import: CSV mapper with validation preview.

### 3. Campaigns
- List: name, offer, status, sent/opened/replied, mailboxes, progress bar.
- **Campaign builder:** stepper —
  1) Basics (name, offer/service being sold, timezone),
  2) Audience (attach leads by filter/selection; shows how many are verified+ready),
  3) **Sequence** (steps: initial + follow-ups; per step: AI or fixed, delay, trigger; live AI preview),
  4) Sending (mailbox pool selector w/ health, daily caps, random interval, sending windows),
  5) Review + **launch validation** (blocks if no healthy mailbox / suppression conflicts).
- Running campaign view: per-step funnel, per-lead state table, pause/resume/stop, live health.

### 4. Inbox (unified)
- 3-pane: thread list (label filter chips: Interested/Meeting/Pricing/Referral/Not-interested/Spam) │
  conversation │ context panel.
- Each thread: **AI label badge + confidence**, **AI summary**, **suggested reply** (one-click insert, editable).
- Reply composer with mailbox selector, snippets, and "convert to deal" / "book meeting" actions.
- Bulk triage; keyboard nav (j/k, e to archive, r to reply).

### 5. CRM
- **Kanban pipeline** (drag between stages: lead→…→won/lost) + optional table view.
- Deal drawer: value, stage history, linked emails/threads/meetings/proposals, activity timeline, notes.
- Quick filters by owner/stage/value; forecast summary.

### 6. Analytics
- Tabs: Overview · Deliverability · Campaigns · Funnel · Revenue.
- Deliverability: per-mailbox trends (sent/bounce/complaint), health score over time, blacklist alerts, warmup progress.
- Charts: Recharts (line/bar/funnel), date-range + comparison. Export CSV.

### 7. Mailboxes
- Card grid: each mailbox = provider icon, email, health ring, cap used today, warmup state, status.
- Connect flow: choose Google/M365 (OAuth) or SMTP/IMAP (form). Test button (SPF/DKIM/DMARC + send test).
- Warmup controls + history chart.

### 8. Settings
- Sections: Profile · Workspace · **Compliance** (company address, unsubscribe copy, sending windows,
  suppression list manager, GDPR export/delete) · Team (later) · Integrations · Billing (later) · API keys (later).
- **Compliance section is prominent**, not buried — it gates legal safety.

### 9. Admin panel *(later)*
- Users, workspaces, mailboxes/domains overview, credits/subscriptions, audit logs, system health.
- Role-gated (Admin only).

### 10. Onboarding
- First-run wizard: create workspace → connect first mailbox → set compliance identity → import/find first leads
  → build first campaign. Progress persists.

## Cross-cutting UI
- **AI review pattern (reusable):** draft shown with source evidence (the audit facts it used) → Edit / Approve /
  Reject / Regenerate. Cost/model shown subtly. Builds trust, prevents hallucinated sends.
- **Async everywhere:** optimistic UI + job progress toasts/badges (TanStack Query + polling/SSE on `/jobs/:id`).
- **Empty/loading/error states** designed for every screen (skeletons, retry, helpful empties).
- **Notifications:** new replies, mailbox unhealthy, campaign finished, job done.

## Responsive behavior
- **Desktop-first** (it's an operator tool) but graceful down to tablet/mobile:
  - Sidebar → bottom nav / drawer on mobile.
  - Tables → card lists on small screens; horizontal scroll containers for wide data.
  - Kanban → stacked columns with horizontal scroll on mobile.
  - Charts → simplified, scrollable.
- Touch targets ≥ 44px; keyboard-first on desktop.

## Accessibility & theming
- WCAG AA contrast in **both dark (default) and light** themes; theme toggle persists.
- Full keyboard nav, focus rings, ARIA on interactive components, reduced-motion honored (Framer Motion).
- Color is never the only signal (health uses icon + label + color).

## Component/state architecture
- **Server state:** TanStack Query (caching, background refetch, optimistic updates, job polling).
- **Forms:** React Hook Form + Zod (shared schemas with backend where possible).
- **Local/UI state:** lightweight store (Zustand) for shell, filters, command palette.
- **Feature-folder structure:** `features/{leads,campaigns,inbox,crm,analytics,mailboxes,settings}` each with
  `components/ hooks/ api/ types/`.
